Introduction

Microsoft azure is a cloud computing service created by Microsoft, it is Infrastructure as a Service(IaaS) and Platform as a Service(PaaS) that can be used for storage, networking, computing and much more. The Developers have the option to select from the languages such as C#, PHP, HTML5, Java, Python, and Node.ja to write their code and use easy tools platform-specific Azure SDKs and Visual studio to start immediately. The readers should have a basic knowledge about azure, since this book focuses on Azure web apps and not on essential aspects of the Microsoft Azure platform.

Introduction to azure apps service plans

An App service plan allows us to define the region and pricing tier which can be shared accross the app service types, which are Web Apps, Mobile Apps, Logic Apps, and API Apps. The App services are categorised into five types:

  1. Free The free tier is considered for evaluation purpose; that is why it is free. Web apps in this tier share machine resouces with other web apps from other azure subscribers. Since the resources are shared, you have limited daily compute and network bandwidth limits.
  2. Shared The shared tier also shares machine resources with other web apps, but the daily compute and network bandwidth limits are increased. This tier is intended for low-traffic sites and enables support for a few features, such as custom domains and the ability to scale out to multiple shared instances.
  3. Basic The basic tier gives the dedicated machine instances to host your web app and, therefore, has no daily resource limits. In this tier, you are able to scale up the size of your dedicated virtual machine to increase the number of cores and RAM. This tier enables additonal Azure Web App features and also is backed by SLA
  4. Standard The standard tier has all the features of the basic tier, and it enables all the features available to Azure web apps. Some of the features such as automated backups, autoscale,and more are enabled in this tier
  5. Premium The premium tier provides same set of feature like the standard tier but, with some extra capacity. It also enables BizTalk integration capabilities.

Creating an Azure Web App using the Azure portal

In Visual Studio, create a project by selecting File > New > Project.
In the New Project dialog, select Visual C# > Web > ASP.NET Core Web Application.
Name the application myFirstAzureWebApp, and then select OK.

You can deploy any type of ASP.NET web app to Azure. For this quickstart, select the MVC template, and make sure authentication is set to No Authentication
. Select Enable Docker Compose support
. Select OK

.

From the menu, select Debug > Start without Debugging to run the web app locally.

Publish to Docker Hub

In the Solution Explorer, right-click the myFirstAzureWebApp project and select Publish

The publish wizard is automatically launched. Select Container Registry > Docker Hub > Publish.

Supply your Docker Hub account credentials and click Save.
Wait for the deployment to complete. The Publish page now shows the repository name that you will use later in App Service.
